## Hard gates (all must pass to advance)

1. `cargo check --workspace`
2. `cargo clippy --workspace -- -D warnings`
3. `cargo test --workspace`
4. `cargo deny check`
5. BAD_RUST audit (`scripts/audit.sh`) — zero `BLOCKING` findings
6. Proof-lane routing — every changed path resolves to ≥1 lane in `agent/proof-lanes.toml`

## Quality score (drives tiebreak)

```

      +  3*test_coverage_pct
      +  2*proptest_density
      +  1*doc_coverage_pct

```

Higher is better. The outer loop advances when all hard gates pass AND `score` improved AND no MUST-AC regressed.

## Things to never do

(Excerpted from `rules/bad-rust.md`. Full catalog at `~/.claude/skills/autobuilder/rules/bad-rust.md`.)

- Don't add `clone()`, `Arc<Mutex<_>>`, `Box::leak`, `Rc<RefCell<_>>`, `static mut`, `unsafe`, or `transmute` to satisfy the borrow checker.
- Don't replace clear ownership with shared mutable state.
- Don't broaden a precise error type to `anyhow::Error` in a public lib API.
- Don't weaken or `#[ignore]` a test to match generated output.
- Don't `unwrap()` / `expect()` on user/network/file/env/CLI input. Use `?`.
- Don't add a `todo!()`, `unimplemented!()`, `panic!("should not happen")`, or placeholder branch in a reachable path.
- Don't add a dependency in the same iteration as a behavior change.
- Don't modify files outside `src/` without an amendment request.

## When stuck

If you keep proposing fixes that the outer loop reverts, stop. Read the last 5 rows of `results.tsv`. Are you re-trying the same hypothesis with slight rewording? Pivot to a different hypothesis or write `intent_card_amendment_request.json`.

If `retry_count` on the latest FailureCapsule is 3 and you're crashing the same way, write an amendment — the AC may be impossible under the current constraints.
